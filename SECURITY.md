# Security Policy / Política de Seguridad

## Supported Versions / Versiones Soportadas

**EN:** Only the latest minor release receives security fixes. Please update before reporting.

**ES:** Solo la última versión menor recibe correcciones de seguridad. Por favor, actualiza antes de reportar.

| Version | Supported |
|---------|-----------|
| latest  | ✅        |
| < latest | ❌       |

## Reporting a Vulnerability / Reportar una Vulnerabilidad

**EN:**
Please **do not** open a public GitHub issue for security vulnerabilities.

Report security issues privately via **[GitHub Private Security Advisories](https://github.com/nexgou/server/security/advisories/new)**.

When reporting, include:
- A clear description of the vulnerability
- Steps to reproduce or a proof-of-concept
- The potential impact and affected versions
- Any suggested mitigations (optional)

We aim to acknowledge reports within **72 hours** and to release a fix within **14 days** for critical issues.

---

**ES:**
Por favor, **no** abras un issue público de GitHub para vulnerabilidades de seguridad.

Reporta los problemas de seguridad de forma privada a través de **[GitHub Private Security Advisories](https://github.com/nexgou/server/security/advisories/new)**.

Al reportar, incluye:
- Una descripción clara de la vulnerabilidad
- Pasos para reproducirla o prueba de concepto
- El impacto potencial y las versiones afectadas
- Mitigaciones sugeridas (opcional)

Nos comprometemos a acusar recibo en **72 horas** y a publicar una corrección en **14 días** para problemas críticos.

## Disclosure Policy / Política de Divulgación

**EN:** We follow coordinated disclosure. Once a fix is released, we will publish a security advisory crediting the reporter (unless anonymity is requested).

**ES:** Seguimos la divulgación coordinada. Una vez publicada la corrección, publicaremos un aviso de seguridad reconociendo al reportero (a menos que solicite anonimato).
